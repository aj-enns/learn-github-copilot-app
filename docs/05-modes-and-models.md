# 05 · Modes, models & reasoning effort

**Time:** ~2 min on screen · **Goal:** give attendees a mental model for the knobs they'll reach
for in every session. This is a recap slide more than a demo.

## Session modes — how much autonomy

| Mode | Who drives | Use it when |
| --- | --- | --- |
| **Interactive** | You and the agent, turn by turn | You want tight steering or the task is ambiguous |
| **Plan** | Agent proposes a plan, you approve, then it executes | You want to validate scope/approach before any code |
| **Autopilot** | Agent works fully autonomously — writes code, runs tests, iterates | The task is well‑defined and low‑risk |

You can **change the mode at any time** during a session — start in Plan, drop to Interactive to
steer, finish on Autopilot.

## Models & reasoning effort

- Pick the **model** per session from the model picker. If you've configured **bring your own key
  (BYOK)**, your provider's models appear here too.
- **Reasoning effort** trades time for depth: higher effort gives the agent more room to think
  through hard problems, but takes longer.
- Both are changeable mid‑session.

## Matching the knobs to the task (the efficiency point)

- **Light model + Autopilot** for straightforward, well‑scoped changes (e.g. add a small endpoint).
- **High‑capability model + higher effort** for complex debugging, design decisions, and
  multi‑step work.
- **Plan mode** to validate the approach before spending tokens on code.
- **Quick chats** to scope before opening a full session at all.

> Rule of thumb: spend reasoning where the risk is. Don't pay for max effort on a one‑line fix.
